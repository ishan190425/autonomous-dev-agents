## Learnings (L280-L284)

### L280: TRUE Day 1 Transition Protocol (C571)

Blocked-time preparation + structural first checks (R-013) + role independence enables instant execution when blockers clear. Crisis handling is structural, not improvised.

### L281: Historical Documentation Preservation (C574)

Historical docs should preserve original context (what happened, not retroactive edits). Only user-facing docs need updates for naming changes.

### L282: Placeholder Tokens for Volatile Copy (C577)

Pre-written announcement copy should use placeholder tokens (`{{PACKAGE_NAME}}`, `{{CYCLE_COUNT}}`) for volatile values. Kit at C557 used literal `@ada/` but packages published as `@ada-ai/`.

### L283: npm Stats Propagation Delay (C578)

npm download statistics have 24-48h delay for new packages. T+24h may miss downloads; T+48h more reliable.

### L284: Early Launch Runway ‚Üí Feedback, Not Scope (C580)

When launches happen early, use the extra runway for feedback collection, not scope expansion. Feb 14 launch (vs planned Feb 24) creates 14-day pre-Sprint 2 runway ‚Äî validate priorities with real user data.

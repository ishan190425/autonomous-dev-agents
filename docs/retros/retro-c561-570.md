## What Could Improve

### 1. Pre-publish Import Verification (L279)

CLI broke because `heat.ts` imported `@ada/core` but packages were `@ada-ai/core`. The scope rename happened in `package.json` but not all source files.

**Action:** Add import verification to pre-publish checklist:

```bash
grep -r "from '@<old-scope>" --include="*.ts"
```

### 2. Scope Naming Due Diligence (L278)

`@ada` was unavailable on npm, forcing last-minute rename to `@ada-ai`. This created #140 (doc cleanup needed).

**Action:** Verify npm org/scope availability early in launch planning.

### 3. Publishing Secret Verification (L274)

NPM_TOKEN missing wasn't caught until 19+ hours after release trigger.

**Action:** Pre-launch checklists must explicitly verify each publishing secret exists.

Compression was deferred from C477 to C504 (27 cycles) without explicit tracking. While deferral during launch countdown is acceptable (L227), we should track deferral reasons and target dates.

**Action:** Add "Compression Status" to memory bank header. Track: last compression cycle, target next, deferral reason.

### 2. Protocol Parallelization

Engineering and Product created similar Day 1 protocols independently. Could have been coordinated into a unified template.

**Action:** Create `templates/day1-role-protocol.md` for future launches.

T-0 night cycles (C505-C510) were largely verification without new deliverables. Efficient for risk reduction, but team could use this pattern intentionally.

## üìö Learnings Identified

### L163: Duplicate action logging indicates workflow gap

- **Context:** C404 (Ops) logged nearly identical action to C403 (Engineering) ‚Äî both described Heat Scoring Core Module implementation.
- **Insight:** This could indicate: (a) git rebase/merge confusion during dispatch complete, (b) ada CLI not validating action uniqueness, or (c) role confusion about what counts as "new action." Either way, the dispatch system should catch this.
- **Action:** Consider adding action deduplication warning to `ada dispatch complete`. File issue if pattern recurs.
- **Status:** monitoring

### L164: Pre-decision holding pattern enables Sprint N+1 velocity head start

- **Context:** With Go/No-Go pending and MUST criteria complete, team pivoted to Sprint 2 prep. Engineering shipped Heat core (C403), Design resolved UX questions (C405), Research/Frontier/Product created execution protocols.
- **Insight:** The natural inclination during "waiting" periods is to slow down. But with specs frozen and tests green, this is the safest time to build foundational infrastructure. Sprint 2 starts with ~33% of work already done.
- **Action:** When entering pre-decision holding periods, explicitly shift focus to N+1 sprint scaffolding. Document this as standard operating procedure.
- **Status:** applied (C401-410 demonstrated pattern)

### L165: Execution protocols that consolidate prior research eliminate context-switching

- **Context:** Research C408 consolidated 9 cycles of benchmark research (C148, C268, C278, C298, C308, C309, C328, C348, C378) into single execution protocol. Frontier C409 similarly unified Heat + Memory + Terminal specs.
- **Insight:** Scattered research/spec documents across many cycles create context-switching burden at sprint kickoff. A single consolidated "execution protocol" document gives clear week-by-week timeline, success criteria, and dependencies ‚Äî all in one place.
- **Action:** Before sprint kickoff, each domain should produce an execution protocol that consolidates prior research. Format: timeline, requirements, success criteria, risks.
- **Status:** applied (C408, C409 demonstrated pattern)

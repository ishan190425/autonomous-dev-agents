## Learnings Identified

### L295: Observer Mode is Earned, Not Assumed

**Context:** CEO (C606) validated observer mode after 10/10 role execution with zero intervention.  
**Insight:** Post-launch observer mode works when structural discipline (R-013, retro gates, CLI dogfooding) has compounded through 189+ consecutive cycles. New teams shouldn't assume observer mode works.  
**Action:** Document observer mode preconditions in CEO playbook: minimum consecutive cycles, R-013 compliance rate, retro cadence adherence.  
**Status:** pending

### L296: UX Specifications Before Engineering Prevent Design Debates

**Context:** Terminal Mode UX spec (C605) defined visual separation patterns (box-drawing, color modes, streaming) before Engineering implementation.  
**Insight:** Pre-implementation UX specs with concrete visual examples prevent mid-implementation design debates. Design and Engineering align on output format before code is written.  
**Action:** Add to Design playbook: for CLI-facing features, UX spec with visual examples required before Engineering implementation.  
**Status:** pending

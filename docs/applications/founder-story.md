## The Origin Story

### The Copilot Tax

Every session followed the same pattern:

1. Write a prompt
2. Review the output
3. Catch the hallucinations
4. Fix the integration bugs
5. Test what the AI didn't test
6. Document what the AI skipped
7. Deploy what the AI couldn't access

I was saving time on typing and spending it on coordination. The tools helped with _parts_ of development, but I was still doing all the _meta-work_: planning, integration, quality, operations.

### The Insight

One night, reviewing yet another AI-generated PR that "almost worked," I had the thought:

**"Human dev teams work because of roles, not raw capability."**

A junior developer becomes productive not by knowing everything, but by having:

- A defined role with clear responsibilities
- Playbooks for common tasks
- Access to team context and decisions
- Structured handoffs with other team members
- Someone to check their work

What if AI agents worked the same way?
